# Promptimall: vea analüüs

## Eesmärk

Seda promptimalli saab kasutada olukorras, kus lahendus, kood, arvutus või mõttekäik ei tööta ootuspäraselt. Eesmärk ei ole lasta mudelil kogu töö kasutaja eest ümber teha, vaid mõista, kus viga tekib ja kuidas seda tulevikus vältida.

## Promptimall

```txt
Käitu juhendajana, kes aitab mul viga mõista.

Kontekst:
[lisa lühidalt, mida proovid teha]

Minu lahendus / kood / mõttekäik:
[lisa oma töö]

Veateade või probleem:
[lisa veateade või kirjelda, mis ei tööta]

Oodatud tulemus:
[lisa, mida programm, arvutus või lahendus peaks tegema]

Tegelik tulemus:
[lisa, mis tegelikult juhtub]

Palun:
1. selgita, kus viga tõenäoliselt tekib;
2. selgita, miks see viga tekib;
3. paku parandust;
4. näita, kuidas sarnast viga tulevikus vältida;
5. ära kirjuta kogu lahendust ümber, kui piisab konkreetse vea parandamisest.
```

## Programmeerimise jaoks sobiv variant

```txt
Käitu programmeerimise juhendajana.

Kontekst:
[lisa, mida programm peaks tegema]

Kood:
[lisa kood]

Veateade:
[lisa veateade, kui see on olemas]

Oodatud tulemus:
[lisa oodatud tulemus]

Tegelik tulemus:
[lisa tegelik tulemus]

Palun:
1. leia vea tõenäoline põhjus;
2. selgita, miks see viga tekib;
3. näita minimaalset parandust;
4. ära kirjuta kogu programmi ümber, kui see pole vajalik;
5. selgita, kuidas ma saaksin sarnast viga tulevikus vältida.
```

## Kirjaliku töö või mõttekäigu kontrollimiseks

```txt
Käitu juhendajana, kes aitab mul mõttekäiku kontrollida.

Minu tekst või lahendus:
[lisa tekst või lahenduskäik]

Minu eesmärk:
[lisa, mida soovid tekstiga saavutada]

Palun:
1. leia kohad, kus mõttekäik on ebaselge või loogiliselt nõrk;
2. selgita, miks need kohad vajavad parandamist;
3. paku konkreetseid parandusi;
4. ära muuda minu stiili rohkem kui vajalik;
5. too lõpus välja 2–3 kõige olulisemat soovitust.
```

## Kasutusolukord

Seda prompti saab kasutada näiteks:

- programmeerimisvigade leidmisel;
- SQL-päringute parandamisel;
- arvutuskäigu kontrollimisel;
- kirjaliku töö loogika ülevaatamisel;
- mõistete või argumentide täpsustamisel.

## Soovitus

Mida täpsemalt kirjeldada oodatud ja tegelikku tulemust, seda paremini saab mudel aidata. Ainult lause „see ei tööta” ei anna mudelile piisavalt infot, et viga täpselt analüüsida.
